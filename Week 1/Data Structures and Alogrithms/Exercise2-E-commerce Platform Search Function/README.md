Exercise 2: E-commerce Platform Search Function

Big O notation is used to measure the efficiency of an algorithm as the input size increases. It helps us understand how the running time of an algorithm grows with larger datasets.

Linear Search:
- Best Case: O(1) when the element is found at the first position.
- Average Case: O(n)
- Worst Case: O(n) when the element is at the last position or not present.

Binary Search:
- Best Case: O(1) when the element is found at the middle position.
- Average Case: O(log n)
- Worst Case: O(log n)

Comparison:
Linear search checks elements one by one and is suitable for small datasets.
Binary search is much faster for large datasets but requires the data to be sorted.

Conclusion:
For an e-commerce platform containing thousands of products, binary search is a better choice because it reduces the search time significantly compared to linear search.
